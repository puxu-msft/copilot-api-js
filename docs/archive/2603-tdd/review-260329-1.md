## 发现 1：`02-infrastructure.md` 对 `test:ci` 的说明与当前 `package.json` 冲突

严重级别：中

### 问题

`docs/tdd/02-infrastructure.md` 前文已经正确写到：

- 当前 `test:backend` / `test:ci` 包含 `unit/component/contract/integration/e2e`

但同一文档在 CI 命令示例里又写：

```yaml
- bun run test:ci  # 单元 + 组件 + 契约 + 集成（不含 E2E、HTTP、WS）
```

这和当前 `package.json` 不一致。现在仓库里：

- `test:backend = bun test tests/unit/ tests/component/ tests/contract/ tests/integration/ tests/e2e/`
- `test:ci = npm run test:backend`

因此当前的 `test:ci` 实际上**包含 `tests/e2e/`**，并不是文档里写的“不含 E2E”。

### 依据

- `docs/tdd/02-infrastructure.md:35`
- `docs/tdd/02-infrastructure.md:448`
- `package.json:42-44`

### 风险

这会直接影响后续 CI 设计讨论：

- 如果团队按文档理解，会以为 `test:ci` 仍是“快速后端测试”
- 但按当前脚本实际运行，它已经把 `tests/e2e/` 也算进去了

这种冲突不会阻塞文档整体方向，但会让后续落地时对 CI 时间预算和测试分层产生误判。

### 建议

把口径统一成其中一种：

1. 如果保留当前 `package.json` 结构，就把文档改成：
   - `test:ci` 当前包含 `tests/e2e/`
   - 后续再决定是否拆分成“不含 E2E 的 backend CI”与“单独 E2E”

2. 如果目标是让 `test:ci` 不含 E2E，就明确把这点写成“计划调整”，而不是把尚未发生的行为写成现状
